CATEGORIA 1: BUGS DE UI / RENDERIZADO
--------------------------------------------------------------------------------

[  ] 1.1 - Fix Superposicion del Contenedor (Overlay)
     Descripcion: El area principal del juego se desplaza hacia arriba y tapa
     parcialmente la barra de navegacion superior, impidiendo ver el saldo
     y los botones de menu.
     Prioridad: ALTA

[  ] 1.2 - Fix Sprite Clipping de la Gallina
     Descripcion: El grafico de la gallina aparece por encima de la barra de
     navegacion. Los elementos moviles del juego deben quedar siempre por
     debajo de la interfaz global (z-index correcto).
     Prioridad: ALTA

--------------------------------------------------------------------------------
CATEGORIA 2: OPTIMIZACION DEL LAYOUT / FLUJO DE USUARIO
--------------------------------------------------------------------------------

[  ] 2.1 - Subir el Boton CTA (Place Bet) por encima del Fold
     Descripcion: El boton "Place Bet" queda oculto debajo del limite visual
     de la pantalla. El usuario debe hacer scroll para encontrarlo, lo cual
     rompe el ritmo de juego y reduce engagement. El ciclo completo
     (ver juego -> decidir apuesta -> apostar) debe ocurrir en una sola
     vista sin scroll.
     Prioridad: ALTA

[  ] 2.2 - Compactar Botones de Dificultad
     Descripcion: Reducir el tamano de los botones de dificultad (Easy, Medium,
     Hard, Extreme) o ponerlos en una sola fila horizontal compacta. Esto
     libera espacio vertical para que el boton de apuesta sea visible
     inmediatamente sin perder la seleccion de un solo toque.
     Prioridad: ALTA

[  ] 2.3 - Reducir Carga Cognitiva del Panel de Apuestas
     Descripcion: Simplificar la interfaz para que el usuario se enfoque en
     la accion principal (apostar). Considerar ocultar opciones secundarias
     en menus colapsables. Referencia: Rainbet muestra menos opciones
     simultaneamente para mantener el foco en el boton de jugar.
     Prioridad: MEDIA

--------------------------------------------------------------------------------
CATEGORIA 3: DISENO VISUAL
--------------------------------------------------------------------------------

[  ] 3.1 - Redisenar Vehiculos/Carros
     Descripcion: Refinar el diseno visual de los vehiculos siguiendo los
     criterios de diseno de Jean. Mejorar sprites/graficos de los carros
     para mayor calidad visual.
     Prioridad: MEDIA

--------------------------------------------------------------------------------
CATEGORIA 4: SISTEMA DE AUTOBET (NUEVA FUNCIONALIDAD)
--------------------------------------------------------------------------------

[  ] 4.1 - Implementar Sistema de Autobet
     Descripcion: Reemplazar o complementar el Auto Cash-Out actual con un
     sistema completo de Autobet al estilo Rainbet. Debe incluir:
     - Configuracion de cantidad de apuestas automaticas (Number of Bets,
       incluyendo opcion "infinito")
     - Ajuste automatico On Win (ej. aumentar apuesta X% al ganar)
     - Ajuste automatico On Loss (ej. aumentar apuesta X% al perder)
     - Stop On Profit: parada automatica al alcanzar ganancia maxima
     - Stop On Loss: parada automatica al alcanzar perdida maxima
     Prioridad: MEDIA

[  ] 4.2 - UI Colapsable para Autobet
     Descripcion: Dado que el Autobet tiene muchas opciones, la interfaz debe
     ser colapsable o estar muy bien organizada para no ocupar demasiado
     espacio vertical. Considerar un panel expandible/colapsable.
     Prioridad: MEDIA

--------------------------------------------------------------------------------
CATEGORIA 5: FEED SOCIAL / DATOS SOCIALES
--------------------------------------------------------------------------------

[  ] 5.1 - Implementar Feed Global de Jugadas
     Descripcion: Agregar un feed de jugadas globales (de todos los jugadores)
     con categorias como "Big Wins" y "Lucky Wins". Actualmente solo se
     muestran jugadas individuales del propio jugador. El feed global
     fomenta comunidad y competencia entre jugadores.
     Prioridad: MEDIA

--------------------------------------------------------------------------------
CATEGORIA 6: DESCRIPCION DEL JUEGO Y REGLAS
--------------------------------------------------------------------------------

[  ] 6.1 - Agregar Seccion de Descripcion/Reglas del Juego
     Descripcion: Integrar una pestana o seccion de "Description" que explique
     la mecanica del juego, reglas, como funciona el multiplicador y la
     cantidad de obstaculos por dificultad. Esto reduce la curva de
     aprendizaje y da confianza al jugador.
     Prioridad: ALTA

[  ] 6.2 - Agregar Seccion de Fairness (Transparencia / Provably Fair)
     Descripcion: Implementar una pestana de "Fairness" que explique al
     usuario como funciona el sistema de justicia del juego (Provably Fair).
     Mostrar seeds, verificacion, y como se determinan los resultados.
     Esto genera confianza y transparencia.
     Prioridad: ALTA
